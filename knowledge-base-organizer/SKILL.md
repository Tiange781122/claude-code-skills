## 功能概述

本技能可自動化執行知識庫整理任務，遵循 CLAUDE.md 的規範：

1. **掃描並診斷**：檢查知識庫中的結構問題
2. **分析文檔**：根據檔案名稱和內容判斷應屬分類
3. **建議方案**：生成整理方案供用戶確認
4. **自動整理**：建立資料夾、移動檔案、驗證結果
5. **生成報告**：記錄整理前後的結構對比

---

## 核心原則

### 容器原則（必須遵守）
- **容器資料夾**：只包含子資料夾（無 .md 卡片）
- **葉子資料夾**：只包含 .md 卡片（無子資料夾）
- **禁止混放**：同一資料夾不能既有子資料夾又有 .md 檔案

### 編號系統
- **階層式編號**：X.Y.Z.N...（逐層遞增）
- **保持連續**：超過 7 個分類時新增層級
- **清晰命名**：資料夾名稱與內容強相關

### 葉資料夾優先原則
1. **優先放入現有葉資料夾**：若已有符合的葉資料夾，直接移入
2. **無適合時新建分類**：根據母資料夾編號系統建立新子資料夾
3. **命名即知識體**：新資料夾名稱應直接體現**協議/技術名稱**，而非籠統的分類詞
   - ✅ 好：`2.1.2.5.CDP思科發現協議`、`2.1.2.7.DHCP動態主機配置`
   - ❌ 避免：`2.1.2.1.協議類`、`2.1.2.2.服務配置`、`2.1.2.3.基礎概念`

## 掃描與診斷

### 檢查項目

**1. 容器混放檢測**
```
❌ 違規結構
2.1.交換器/
├─ 2.1.1.Aruba/           ← 子資料夾
├─ 2.1.2.Cisco/           ← 子資料夾
└─ 某文件.md              ← 混放的 .md 檔案（違規！）

✅ 正確結構
2.1.交換器/（容器）
├─ 2.1.1.Aruba/（容器或葉）
└─ 2.1.2.Cisco/（容器或葉）
```

**2. 編號系統檢查**
- 編號是否連續（不應跳號）
- 是否超過 7 層級而未新增層級
- 資料夾名稱前綴是否正確

**3. 檔案位置檢查**
- 是否存在在非葉資料夾中的 .md 檔案
- 檔案名稱是否需要去除編號前綴（移動到子資料夾後）

**4. 命名清晰度檢查**
- 資料夾名稱是否直接體現協議/技術/知識體
- 避免籠統分類詞（如「協議」、「配置」、「基礎概念」）
- 名稱應讓使用者一眼看出內容

## 使用流程

### 基本用法

```
/kbo 掃描並整理 <obsidian-vault-path>/2.網路基礎設施/2.1.交換器
```

### 流程步驟

**第 1 步：掃描診斷**
```
掃描指定路徑
↓
檢查容器混放
檢查編號系統
檢查檔案位置
↓
輸出問題清單
```

**第 2 步：分析分類**
```
檢查容器混放的 .md 檔案
↓
根據檔案名稱判斷應屬分類
分析現有葉資料夾
↓
提出分類建議
```

**第 3 步：用戶確認**
```
展示分類方案
↓
列出將新建的資料夾
列出將移動的檔案
↓
等待用戶確認
```

**第 4 步：自動整理**
```
建立新資料夾（遵循編號系統）
↓
移動檔案到目標位置
去除不必要的編號前綴
↓
驗證最終結構
```

**第 5 步：輸出報告**
```
整理前結構樹
整理後結構樹
變更摘要（新增資料夾、移動檔案數量）
```

## 限制與注意

- **備份提醒**：整理前應確保已備份知識庫
- **不刪除檔案**：僅進行分類和移動，不刪除任何內容
- **不修改內容**：不改動 .md 檔案的正文，僅調整位置和名稱
- **手動確認**：大規模整理前應要求用戶確認
- **增量整理**：可針對特定資料夾進行局部整理

### 整理方法論

**步驟 1：備份原結構**
```bash
cp -r "5.專業知識基礎" "5.專業知識基礎.backup_$(date +%Y%m%d_%H%M%S)"
```

**步驟 2：根據檔名掃描與分類**
- 首先遍歷所有 .md 檔案，按檔名判斷主題
- 若檔名模糊（如「介紹.md」），讀取前 50 行內容判斷
- 建立檔案→目標資料夾的映射表

**步驟 3：建立新的 3 層結構**
```bash
# 先清空原資料夾，重新建立
rm -rf 5.專業知識基礎
mkdir -p 5.專業知識基礎/5.1.網路協定/5.1.1.BGP邊界閘道協定
mkdir -p 5.專業知識基礎/5.2.資訊安全/5.2.1.PKI公鑰基礎設施
# ...（依次建立所有 26 個二級資料夾）
```

**步驟 4：使用 find + while 循環移動檔案**
```bash
find "$BACKUP/5.1.網路協定/5.1.1.BGP路由" -name "*.md" -type f | while read f; do
  cp "$f" "$NEW_BASE/5.1.網路協定/5.1.1.BGP邊界閘道協定/"
done
```

**步驟 5：驗證與清理**
- 統計新結構中的檔案數，與備份對比
- 檢查是否有遺漏或重複
- 驗證所有二級資料夾中只有 .md 檔案（無子資料夾）
- 檢查 wikilink 是否需要更新（通常自動更新）
- 刪除備份（確認後）

### 關鍵要點

1. **3層結構的優勢**
   - 層級淺，導航快：最多 3 次點擊到達檔案
   - 清晰有序：編號 5.1-5.7 一目瞭然
   - 易於擴展：新增知識體只需加新編號

2. **命名即知識體的實踐**
   - ❌ 舊：5.2.1.PKI與憑證管理/5.2.1.1.PKI基礎概念
   - ✅ 新：5.2.1.PKI公鑰基礎設施（直接體現技術名稱）
   - 一眼看出資料夾內容，無需進入查看

3. **日期前綴的保留**
   - 檔名如 `2026-04-21_OSI七層模型...md` 保留日期前綴
   - 便於追蹤文檔建立/更新時間
   - 日期在檔名而非資料夾名，保持命名一致性

4. **自動化處理的限制與人工操作**
   - 自動化：根據檔名建立資料夾、複製檔案、驗證
   - 人工確認：複雜的內容判斷、編號決策
   - 備份 + 驗證：降低整理風險

### 整理成果
- ✅ 編號系統：5.1-5.7 完全連續，無跳號
- ✅ 容器原則：0 個混放資料夾
- ✅ 文檔保留：104 個檔案完整遷移
- ✅ 連結更新：Obsidian 自動更新 wikilink
- ✅ 時間成本：備份+整理+驗證 < 30 分鐘

## 進階主題：大規模整理的自動化

### 整理流程的自動化 Bash 腳本

對於 100+ 個檔案的整理，以下腳本模式可加速處理：

**模式 1：按檔名規則批量移動**
```bash
# 例：移動所有包含 "BGP" 的檔案
find "$BACKUP" -name "*BGP*" -type f | while read f; do
  cp "$f" "$BASE/5.1.網路協定/5.1.1.BGP邊界閘道協定/"
done
```

**模式 2：按資料夾樹狀結構遞迴複製**
```bash
# 例：複製整個子樹到新位置，並展平結構
find "$BACKUP/5.1.網路協定/5.1.1.BGP路由" -name "*.md" -type f | while read f; do
  cp "$f" "$BASE/5.1.網路協定/5.1.1.BGP邊界閘道協定/"
done
```

**模式 3：驗證整理成果**
```bash
# 驗證檔案數是否一致
backup_count=$(find "$BACKUP" -name "*.md" -type f | wc -l)
new_count=$(find "$BASE" -name "*.md" -type f | wc -l)
[ $backup_count -eq $new_count ] && echo "✅ 全部檔案已複製" || echo "❌ 遺漏檔案"
```

### 處理邊界情況

1. **遺漏檔案的追蹤**
   ```bash
   comm -23 \
     <(find "$BACKUP" -name "*.md" | sed 's|.*/||' | sort) \
     <(find "$BASE" -name "*.md" | sed 's|.*/||' | sort)
   ```

2. **重複檔案的檢測**
   ```bash
   find "$BASE" -name "*.md" | sed 's|.*/||' | sort | uniq -d
   ```

3. **容器混放的檢測**
   ```bash
   for dir in "$BASE"/5.*; do
     md_count=$(find "$dir" -maxdepth 1 -name "*.md" | wc -l)
     sub_count=$(find "$dir" -maxdepth 1 -type d | wc -l)
     sub_count=$((sub_count - 1))
     [ $md_count -gt 0 ] && [ $sub_count -gt 0 ] && echo "❌ 混放: $dir"
   done
   ```
