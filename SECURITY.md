# 🔐 安全政策 (Security Policy)

## 報告資安問題

如果你發現此倉庫中的任何資安漏洞或隱私問題，**請不要**在公開的 Issue 或 Pull Request 中發布。

### 正確的報告方式

請發送電子郵件至：**ab781122@gmail.com**

包含以下信息：
- 漏洞描述
- 受影響的文件或代碼位置
- 潛在影響範圍
- 建議的修復方案（如果有）

### 安全承諾

1. **隱私保護** — 所有敏感信息（API 密鑰、密碼、Token）嚴格保密，不會在程式碼或測試用例中硬編碼
2. **客戶信息** — 測試用例中使用虛構客戶名稱，避免暴露真實業務信息
3. **依賴更新** — 定期更新依賴包並關注安全公告
4. **程式碼審查** — 所有 Pull Request 都經過資安審查

## 已知安全考量

### 1. 測試數據管理
- ✅ 所有客戶信息已匿名化為通用公司名稱（「科技集團 A」、「汽車公司 B」等）
- ✅ 測試用例中的地點為虛構或通用描述

### 2. 認證信息
- ✅ 不包含任何 API 密鑰、OAuth Token、密碼
- ✅ `.gitignore` 已配置排除 `.env` 和 `secrets.*` 文件

### 3. 敏感文件排除
```
.env                    # 環境變數
.env.local
secrets.json            # 密鑰配置
*.key                   # 私鑰
*.pem                   # 憑證
```

## 貢獻者安全指南

如果你為此專案貢獻代碼，請遵守：

1. **不提交敏感信息**
   - ❌ 真實 API 密鑰、密碼、認證 token
   - ❌ 真實客戶名稱、地址、聯絡信息
   - ❌ 內部系統 IP 地址、域名
   - ✅ 使用虛構名稱和示例數據進行測試

2. **使用環境變數**
   ```bash
   # ✅ 正確做法
   API_KEY=${EXTERNAL_API_KEY}
   
   # ❌ 錯誤做法
   API_KEY="sk-1234567890abcdef"
   ```

3. **Review 前檢查**
   - 確認沒有洩露個人或客戶信息
   - 確認 `.gitignore` 已正確配置
   - 使用 `git diff` 檢查敏感文件
