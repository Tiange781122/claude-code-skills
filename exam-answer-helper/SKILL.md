## 功能流程

當用戶提供考題時，你將：

1. **分析題目類型**：判斷是選擇題、計算題、論述題、填空題還是其他類型
2. **處理語言**：
   - 如果是中文：直接解答
   - 如果是英文：全面逐詞分析（題目、選項、核心概念都包含）
3. **🔍 搜尋官方文件**（必須）：
   - 根據題目識別相關廠牌/標準（Cisco、Fortinet、Microsoft、IETF 等）
   - 搜尋該廠牌的官方文件、技術指南、配置手冊作為依據
   - 使用 `search-strategy` skill 推薦的工具（gemini cli 優先 + WebSearch 回退）
   - 確保答案基於官方資訊，而非通用知識
4. **提供答案**：清晰簡潔地給出正確答案（附上官方文件根據）
5. **詳細解釋**：用適合初級工程師/大學程度的語言解釋：
   - 為什麼這是正確答案（引用官方資訊）
   - 涉及的核心概念
   - 常見的誤解或陷阱
6. **選項分析**（僅限選擇題）：逐一解釋每個選項，說明為什麼正確或錯誤
7. **英文逐詞分析**（英文題目必備）：對題目、選項和核心詞彙進行逐詞分析

**完整版** — 包含完整逐詞分析

```markdown

  A project objective / 一個專案目標

  the association time / 關聯時間
  to the different access points / 到不同的存取點
  as mobile devices / 當行動設備
  move around the office. / 在辦公室移動時。

  The ideal solution / 理想的解決方案
  must cover / 必須涵蓋 / 必須支持
  numerous devices / 許多設備
  and device types. / 和設備類型。

  What must be configured? / 必須配置什麼？

**逐詞分析：**

| 短語/子句 | 中文翻譯 |
|---------|---------|
| project objective | 專案目標 / 項目目標 |
| minimize the association time | 最小化關聯時間 / 縮短關聯時間 |

**逐詞分析：**

**中文翻譯：**[完整中文翻譯]

- **D**：[1 句話說明為什麼錯]
```

### 逐詞分析規則

1. **短語分割**：
   - 按邏輯單位（短語、子句）分割，不按單字
   - 每行一個短語，保持自然閱讀節奏
   - 複合短語用「+」連接，表示語法上的組合

2. **翻譯呈現**：
   - 主翻譯（最常用） / 替代翻譯 1 / 替代翻譯 2
   - 多個翻譯用「/」分隔，不超過 3 個
   - 優先選擇符合題目語境的翻譯

3. **表格格式**（推薦）：
   - 左欄：英文短語
   - 右欄：中文翻譯（含多個選項）
   - 便於掃讀和對照

4. **中文翻譯**：
   - 完整繁體中文，1-2 行
   - 保留原文結構和邏輯
   - 確保自然流暢

## 解釋原則

在解釋時，遵循以下原則，確保初級工程師和大學生都能理解：

1. **直奔核心**：快速給出答案和核心要點，避免冗長前言
2. **簡潔清晰**：用簡明語言，三言兩語說清楚，避免冗長論述
3. **要點列舉**：用 bullet points 或簡短段落，易於快速掌握
4. **必要舉例**：只舉最關鍵的例子，不需詳盡說明
5. **選項對比**：簡潔說明為什麼某選項對/錯（一句話為佳）
6. **篇幅控制**：整個解答應控制在 50-100 行以內，確保複習效率

## 與其他 Skills 的協作

- **search-strategy**（強制使用）：搜尋官方文件時必須使用此 skill 推薦的工具策略
  - 優先使用 gemini cli（適合大量技術文件）
  - 回退至 WebSearch（網路可用資源）
  - 並行搜尋多個廠牌官方資源
- **obsidian-note-writer**：新建文章時全面遵循其命名、Frontmatter、Mermaid 圖表、驗證清單等規範
- **obsidian-markdown**：編輯既有筆記時可使用其工具（wikilink、embeds 等）

## 必須要求

- **強制搜尋官方文件**：每次解題前必須搜尋該廠牌/標準的官方資源，不能僅依賴通用知識
- **廠牌識別**：根據題目快速識別相關廠牌（Cisco、Fortinet、Microsoft、Oracle、AWS 等）

## 限制

- 不保存已回答的題目歷史（每次獨立解答）
- 英文關鍵詞分析僅限英文題目（中文題目跳過）
- 關鍵詞選擇原則：優先難懂的專業術語、複合短語、易混淆概念
- 分析格式：逐詞逐短語，並列出主要中文翻譯（用 / 分隔多個含義）
- 若官方文件搜尋無果，須明確說明「找不到相關官方文件，以通用知識回答」
