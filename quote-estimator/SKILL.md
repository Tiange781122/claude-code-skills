# 工程師報價評估 Skill

## 目標

根據工程師提供的會議記錄、專案細節或需求說明，快速產生**完整工項報價**，包括：
- 工時估算 + 費用計算
- 工項分解與執行流程
- 複雜度分析與決策依據
- 需客戶準備的資料清單
- 不確定項目的警示與確認清單

## 核心計費邏輯

### A. 前置作業（公司內部 Lab，平日白天）
前置工時始終按 **1.0 係數**（不加成），包含設備初始化、設定測試、文件製作等。

**基礎前置工時：2 天**

### B. 複雜度係數（疊加在前置工時上）
| 複雜度因素 | 增加工時 | 說明 |
|-----------|--------|------|
| **HA 架構** | +1 天 | Active-Passive 或 Active-Active 高可用設定 |
| **IPsec VPN** | +0.3 天／條 | 每條 VPN tunnel 設定時間 |
| **Firewall Policy** | +0.1 天／10 條 | 每 10 條防火牆規則 +0.1 天 |

### C. 現場施工（現場上架、測試、驗證）
現場工時根據時間類型乘以時間係數：

| 施工時間類型 | 時間係數 | 說明 |
|-----------|--------|------|
| **平日白天** | ×1.0 | 09:00-17:00 標準上班時間 |
| **假日** | ×1.5 | 週末或國定假日 |
| **晚上** | ×1.5 | 下班後（18:00 後）或凌晨 |
| **臨時插隊** | ×1.5 | 工程師需臨時調度其他工作 |

### 最終公式
```
前置工時 = 2 天 + HA(0/1) + IPsec數×0.3 + (Policy數÷10)×0.1
           （平日白天，係數 1.0）

現場工時 = [現場上架時間] × 時間係數
           平日白天：×1.0
           假日/晚上/其他：×1.5

總工時 = 前置工時 + 現場工時
預估費用 = 總工時 × 8,000 元
```

## Step 1：解析輸入文本

從用戶提供的會議記錄、專案細節中識別以下關鍵資訊：

### A. 設備與架構
識別目標防火牆型號及架構需求，記錄：
- [ ] 總部防火牆型號及數量
- [ ] HA 架構方式（是否提到 Active-Passive、Active-Active、Clustering）
- [ ] 外點防火牆型號及數量

### B. VPN 需求
識別 IPsec VPN 條數（不包括 SSL/GlobalProtect VPN）：
- [ ] 明確提到的 VPN 條數
- [ ] 或透過「多點互連」推算（如 Full Mesh 3 地 = 2-3 條 VPN）
- **如果沒有明確提到** → 標記「⚠️ IPsec VPN 條數待確認」

### C. 安全規則
識別 Firewall Policy 數量：
- [ ] 現有防火牆政策條數
- [ ] 新增政策條數
- **如果沒有明確提到** → 標記「⚠️ Firewall Policy 數待確認」

### D. 時間與特殊情況
識別施工時間類型：
- [ ] 平日白天（係數 1.0）
- [ ] 假日、晚上、凌晨、臨時插隊（係數 1.5）
- [ ] 其他特殊情況或加成要求

## Step 2：計算工時與費用

根據識別的複雜度因素，分開計算前置和現場工時：

### A. 計算前置工時（平日白天，係數 1.0）
```
前置工時 = 2 天 + HA係數 + IPsec數×0.3 + (Policy數÷10)×0.1
```

**潤泰案例：**
```
- 基礎：2 天
- HA 架構（91G Active-Passive）：+1 天
- IPsec VPN：未明確提及 → 待確認（估計 +0.6~0.9 天）
→ 前置工時 = 2 + 1 = 3 天（平日白天，無加成）
```

### B. 計算現場工時（套用時間係數）
識別現場施工時間，計算現場工時（如上架、測試、驗證等）：

**潤泰案例：**
```
- 現場上架：假日半天 = 0.5 天 × 1.5（假日加成）= 0.75 天
```

### C. 總工時與費用
```
總工時 = 前置工時 + 現場工時（已乘以時間係數）
預估費用 = 總工時 × 8,000 元

潤泰案例：
總工時 = 3 + 0.75 = 3.75 天
預估費用 = 3.75 × 8,000 = 30,000 元
        （加上 IPsec 確認後調整 +0.6~0.9 天 = +4,800~7,200 元）
```

### D. 確認不確定項
若有以下情況，費用需打上 ⚠️ 待確認：
- **IPsec VPN 條數未明確提到** → 給出「估計範圍」
- **Policy 數量未明確提到** → 列為備註
- **其他複雜度因素或現場時間** → 標記不確定項

## 使用示例

### 範例 1：有明確複雜度信息

**輸入**：潤泰會議記錄
```
總部 91G × 2（HA Active-Passive）
工地 50G × 20 台
FortiManager VM 授權
假日上架施工
```

**輸出**（完整版）：
```
### 潤泰集團 防火牆汰換報價
...
| 基礎 | 2 天 | 16,000 元 |
| HA | 1 天 | 8,000 元 |
| 小計 | 3 天 | 24,000 元 |
| 假日施工 ×1.5 | 4.5 天 | 36,000 元 |

⚠️ 待確認：
- IPsec VPN 條數（工地回總部多點連線）
- 工地外點數與現場施工方式（寄送或現場派員）
```

### 範例 2：資訊不完整

**輸入**：簡短的需求描述
```
中壢新增據點防火牆，需要 VPN 回總部
```

**輸出**（完整版）：
```
### 中壢新據點 防火牆部署報價

⚠️ 預估工時（待進一步確認）：
- 基礎：2 天
- HA 架構：未提及（確認後 +1 天）
- VPN 條數：未提及（確認後 +[N]×0.3 天）
→ 初步估計：2～3+ 天，16,000～24,000+ 元

🔍 必須確認事項：
1. 新據點防火牆是否需 HA 架構
2. VPN 互連拓撲（單點或多點）
3. 現有防火牆政策數量
4. 施工時間（平日或假日）
```

## 工作流程摘要

1. **讀取輸入** → 識別關鍵因素（HA、VPN、Policy、現場施工時間）
2. **計算前置工時** → 2 天 + 複雜度係數（HA、VPN、Policy），平日白天無加成
3. **計算現場工時** → 識別現場施工時間類型，套用時間係數（1.0 或 1.5）
4. **總工時 = 前置 + 現場施工工時** → 得出最終估算
5. **標記待確認項** → 複雜度因素不確定時標記 ⚠️
6. **產出完整報價** → 含前置 / 現場分項、工項分解、備註、確認清單
7. **簡化選項** → 若需要，提供一行報價版本
