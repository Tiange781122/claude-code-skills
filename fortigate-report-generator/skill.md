# FortiGate CIS 組態檢視建議書產生器

將 FortiGate 組態檢視的原始筆記整理為結構化的專業建議書，包含 CIS Benchmark 合規表格、風險分級、各項組態檢視結果及行動建議。

## 何時使用

- 完成 FortiGate 現場組態檢視後，需將筆記整理為可交付客戶的正式建議書
- 手上有 CIS Benchmark 檢查結果（tab 分隔或自由格式）與各項組態截圖筆記

## 前置要求

- 一份包含 CIS 檢查結果的 Obsidian 筆記（任意格式皆可）
- 筆記中可能包含 `![[圖片]]` 引用、CLI 指令、條列式觀察紀錄

## 工作流程

### 步驟 1：讀取原始筆記

讀取使用者指定的筆記檔案，識別以下內容區塊：
- CIS Benchmark 檢查項目（含編號、章節、檢查項目、期望值、實際值、合規狀態、改進建議）
- 各項組態檢視紀錄（Interface、DNS、路由、政策、VIP、資安防護、VPN 等）
- 圖片引用（`![[...]]`）
- CLI 指令片段

### 步驟 2：分類 CIS 項目

將每個 CIS 項目依合規狀態分為四類：
- **合規**
- **不合規**
- **需人工確認**
- **不適用**（環境未使用該功能）

再依風險等級分組：

| 風險等級 | 歸類原則 |
|:---:|:---|
| 高 | 涉及未授權存取、明文傳輸、加密強度、過度開放權限（如 WAN 管理服務、HTTP 協定、ALL Service、Trusted Hosts、強加密、TLS 靜態金鑰） |
| 中 | 影響可視性或違反最佳實務（如 USB 自動安裝、Banner、閒置逾時、日誌設定、登入鎖定） |
| 低 | 強化性質（如 Hostname、CPU 過載記錄、CDN、已合規項目） |
| 不適用 | 環境未使用該功能（如未設定 Zone、未部署 FortiAnalyzer、未使用 SNMP Trap） |

### 步驟 3：計算統計數據

統計各合規狀態的數量與佔比，以及各風險等級的項目數。

### 步驟 4：產出建議書

使用以下結構產出 Markdown 報告：

```markdown
# FortiGate 組態安全檢視與調整建議書

| 項目 | 內容 |
|:---|:---|
| 客戶名稱 | {客戶名稱} |
| 設備型號 | {型號} |
| 設備序號 | {序號} |
| 檢視日期 | {日期} |
| 文件版本 | {版本} |
| 文件日期 | {今日日期} |

---

## 目錄
（自動產生各章節連結）

---

## 一、檢視摘要
（合規統計表格 + 風險等級分布表格）

## 二、CIS Benchmark 合規檢視結果
### 2.1 高風險項目
（表格：CIS 編號 | 檢查項目 | 期望值 | 實際值 | 狀態 | 改進建議）
### 2.2 中風險項目
### 2.3 低風險項目（含已合規項目）
### 2.4 不適用項目
（表格：CIS 編號 | 檢查項目 | 不適用原因）

## 三～十一、各項組態檢視
（依原始筆記內容，每個主題一個章節）

## 最末章、總結與後續行動建議
### 建議處理優先順序
（P1 立即處理 / P2 短期處理 / P3 中期規劃 / P4 持續維護）
### 注意事項
```

### 步驟 5：文字風格轉換

對所有文字內容套用以下規則：
- 移除所有 emoji
- 使用正式技術報告用語（如「予以移除」「落實最小權限原則」「縮減攻擊面」）
- 各章節補充風險說明，解釋「為什麼要改」
- 條列式觀察紀錄轉為表格呈現（含發現事項、風險說明、建議處理方式）

### 步驟 6：圖片分析與具體建議

對筆記中每張圖片進行以下處理：

1. **定位圖片檔案**：解析 `![[圖片名稱]]` 引用，在 Obsidian vault 中搜尋對應的圖片檔案路徑。
2. **讀取圖片**：使用 `fs_read` 的 Image mode 讀取圖片內容。
3. **辨識圖片所屬章節**：根據圖片在筆記中的上下文（前後文字、所屬標題），判斷該圖片屬於哪個檢視主題（如 Interface、DNS、防火牆政策、IPS 等）。
4. **分析圖片內容並產出建議**：依據圖片所屬主題，以資安工程師角度分析截圖中可見的組態內容，產出具體且可執行的建議。

分析時應關注的面向（依主題而異）：

| 圖片主題 | 分析重點 |
|:---|:---|
| Interface | 各介面的 IP 配置、allowaccess 設定、啟用之協定、未使用介面 |
| DNS | DNS 伺服器 IP 是否合理、是否有冗餘或失效的設定 |
| 靜態路由 | 路由目的網段是否仍在使用、是否有重疊或衝突 |
| 防火牆政策 | 來源/目的/服務是否過度開放、日誌設定、流量計數、政策排序合理性、UTM Profile 套用狀況 |
| VIP | 是否有閒置 VIP、對外暴露之服務與埠號 |
| Anti-Virus | 授權狀態、掃描模式（proxy/flow）、Protocol 涵蓋範圍 |
| Web/DNS Filter | 分類過濾設定、是否啟用 FortiGuard 分類 |
| Application Control | 阻擋/監控之應用程式類別、高風險應用是否放行 |
| IPS | 使用之 Profile 與特徵碼數量、action 設定（monitor/block） |
| SSL Inspection | 檢查模式（certificate-inspection/deep-inspection）、例外設定 |
| SSL VPN | 來源限制、認證方式、Portal 設定 |
| 網段分割 | VLAN 劃分、各網段間存取控制 |
| FortiAP | 韌體版本、與 FortiGate 相容性、SSID 安全設定 |

5. **撰寫建議文字**：在圖片引用的上方或下方（視排版需要），插入分析結果與具體建議。建議格式為表格或條列，需包含：
   - 從截圖中觀察到的具體設定值或現象
   - 該設定存在的風險或不符合最佳實務之處
   - 具體的修正指令或操作步驟

6. **保留圖片引用**：`![[...]]` 引用與 `horizontal` code block 必須原封不動保留，建議文字作為圖片的補充說明。

### 步驟 7：輸出需人工確認項目之 CLI 驗證指令

若報告中存在「需人工確認」項目，產出可一次貼入 FortiGate CLI 的驗證指令清單，供使用者取得實際值後回報。

各 CIS 項目對應之驗證指令：

| CIS 編號 | 驗證指令 |
|:---:|:---|
| 1.2 | `config system zone` → `show full-configuration` |
| 1.13 | `config system global` → `show full-configuration \| grep admin-https-ssl-versions` |
| 2.2 | `config system global` → `show full-configuration \| grep admin-lockout` |
| 4.2 | `config system snmp sysinfo` → `show full-configuration` |
| 9.1 | `config log eventfilter` → `show full-configuration` |
| 9.2 | `config log fortianalyzer setting` → `show full-configuration` |

輸出格式範例：
```
config system global
    show full-configuration | grep admin-https-ssl-versions
end

config system zone
    show full-configuration
end
```

### 步驟 8：依回報結果更新報告

使用者貼回 CLI 輸出後，依以下規則判定合規狀態並更新報告：

#### 判定規則

| CIS 編號 | 合規條件 | 不適用條件 |
|:---:|:---|:---|
| 1.2 | 所有 Zone 設定 `intrazone deny` | `config system zone` 為空（未設定 Zone） |
| 1.13 | 包含 `tlsv1-3` | — |
| 2.2 | `admin-lockout-threshold` <= 3 且 `admin-lockout-duration` >= 900 | — |
| 4.2 | `trap-free-memory-threshold` >= 20 且 `trap-freeable-memory-threshold` <= 50 | SNMP `status` = `disable`（未啟用 SNMP） |
| 9.1 | 所有項目皆為 `enable` | — |
| 9.2 | `reliable` = `enable` | `status` = `disable`（未部署 FortiAnalyzer） |

#### 更新流程

1. 依判定規則將每個項目重新分類為「合規」「不合規」或「不適用」
2. 合規項目直接併入對應風險等級表格（2.1/2.2/2.3），狀態標示為「合規」
3. 不適用項目移至「不適用項目」區塊，附不適用原因
4. 不合規項目留在原風險等級表格，更新實際值與改進建議
5. 刪除「需人工確認」狀態，報告中不應殘留任何需人工確認項目
6. 重新計算合規統計（數量與佔比）及風險等級分布
7. 更新 P1~P4 行動建議（如有新增不合規項目需納入對應優先等級）
8. 更新注意事項中的不適用項目編號清單

### 步驟 9：儲存報告

儲存至與原始筆記相同的目錄，檔名格式：`{原始檔名}v{版本號}.md`

## 注意事項

1. **圖片分析**：每張圖片都必須實際讀取並分析，產出與該章節高度相關的具體建議。若圖片無法讀取，則標註「圖片無法讀取，建議人工檢視」。
2. **建議必須具體**：不可只寫「建議調整」，必須指出截圖中看到的具體設定值、問題所在、以及修正指令。
3. **CLI 指令格式**：改進建議中的 CLI 指令使用 inline code 格式。
4. **統計必須正確**：合規統計的數量與佔比必須與表格內容完全一致，每次更新表格後重新計算。
5. **不適用 vs 需人工確認**：環境未使用該功能為「不適用」，功能有使用但數值需確認為「需人工確認」。具體判斷：SNMP status=disable 則 4.2 為不適用；FortiAnalyzer status=disable 則 9.2 為不適用；config system zone 為空則 1.2 為不適用。
6. **合規項目歸位**：經人工確認後的合規項目直接併入對應風險等級表格（2.1/2.2/2.3），不另設獨立區塊。
6. **P1~P4 對應風險等級**：P1 對應高風險不合規項目，P2 對應中風險，P3 對應架構性改善，P4 對應低風險與維護項目。
7. **圖片引用保留**：`![[...]]` 引用與 `horizontal` code block 原封不動保留，分析建議作為補充文字插入。
8. **報告語言**：繁體中文。
