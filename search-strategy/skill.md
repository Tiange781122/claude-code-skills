# 搜尋工具策略

資料搜集與網路搜尋時使用 gemini cli，支援兩種執行模式。

---

## 執行模式：由用戶關鍵字決定

| 用戶關鍵字 | 模式 | 說明 |
|-----------|------|------|
| （無特殊關鍵字） | **Claude 模式** | Claude 直接執行，適合快速互動 |
| `g:` 或 `:g` 開頭 | **Gemini 模式（前景）** | Gemini 全程執行任何任務，Claude 只做最終 QA |
| `gb:` 或 `:gb` 開頭 | **Gemini 模式（背景）** | Gemini 背景執行，Claude 下完指令繼續做其他事 |

Gemini 模式適用範圍：**網路搜尋、翻譯、文字整理、資料彙整、寫入知識庫等任何任務**。

### 範例

```
搜尋有狀態防火牆的資料            → Claude 模式
g: 搜尋有狀態防火牆的資料          → Gemini 前景模式（搜尋）
:g 幫我整理這篇文章改成繁體寫入    → Gemini 前景模式（整理/翻譯）
:gb 搜尋最新 CVE 並整理成卡片      → Gemini 背景模式
```

---

## 工具優先級

1. **gemini cli**（首選）：所有需要搜尋或收集網路資訊的場景
2. **WebSearch**（備選）：gemini 資料品質不佳時才使用

## Claude 模式：Prompt 模板

根據任務類型選擇模板，核心原則：**指定格式、限制長度、要求精確**。

### 📖 概念理解 / 技術說明
```
gemini "[主題]的原理、特點與應用場景，以條列方式說明，附對照表（如有比較對象）"
```

### 🔍 即時資訊 / 最新動態
```
gemini "[產品/CVE/事件] 最新資訊，要求：日期、版本號、具體數據，過濾 [年份] 以前的內容"
```

### ⚖️ 比較 / 選型
```
gemini "[A] vs [B] 比較，以表格呈現：[指定欄位]，並給出使用情境建議"
```

### 🛠️ 問題排除 / 操作步驟
```
gemini "[問題描述] 的解決方法，以步驟條列，附常見錯誤與對應處理"
```

### 📋 資料蒐集 / 清單整理
```
gemini "列出 [主題] 的 [清單類型]，格式：表格或條列，限制在 [N] 項以內"
```

### 💡 頭腦風暴 / 方案建議
```
gemini "針對 [需求/問題]，提供 [N] 個解決方案，每個方案說明優缺點與適用條件"
```

## Gemini 模式：執行方式

### 流程：指令優化 → 執行 → QA

```
用戶指令（模糊）
      ↓
Claude 指令優化（補充背景、格式要求、輸出路徑）
      ↓
gemini 執行（搜尋 → 整理 → 自我 QA → 寫入文件）
      ↓
Claude QA（確認文件品質）
```

### Claude 指令優化原則

收到 `g:` 或 `gb:` 指令後，Claude 在發送給 gemini 前先補充：

| 補充項目 | 說明 |
|---------|------|
| **任務背景** | 用途（知識庫卡片 / 報告 / 個人筆記 / 待辦清單）|
| **任務類型** | 搜尋 / 翻譯 / 整理 / 寫入 / 綜合，決定 Gemini 執行重點 |
| **輸出格式** | Obsidian Markdown 結構（frontmatter / 表格 / 條列 / checkbox）|
| **輸出路徑** | 完整目標路徑（若需寫入文件；否則直接輸出給 Claude）|
| **品質要求** | 搜尋任務：不足時自動補充；整理任務：繁體中文、格式正確 |
| **知識庫規範** | 容器原則、標籤格式、繁體中文 |
| **原始內容** | 若用戶有附上文字，直接貼入 Prompt 傳給 Gemini |

### 優化範例

用戶輸入：
```
g: 搜尋眠豆腐的使用心得
```

Claude 優化後發出：
```bash
gemini -p "
任務：搜尋「眠豆腐」床墊的使用心得與相似競品比較
背景：整理為個人知識庫參考卡片

執行步驟：
1. 搜尋使用心得、評價及競品資訊，不足則自動補充搜尋
2. 整理成 Obsidian Markdown，包含：
   - frontmatter（tags 使用繁體中文分類）
   - 品牌概述
   - 產品線評價（條列）
   - 競品比較表（表格）
   - 選購決策指引
3. 寫入：<obsidian-vault>/0.inbox/眠豆腐使用心得與競品比較.md
" --yolo
```

### 背景執行（gb:）

同上，在 Bash tool 加入 `run_in_background: true`，Claude 發出後立即繼續其他工作，完成後自動收通知。

Claude 在 Gemini 模式下只做：
1. **指令優化**（補充背景、格式、路徑）
2. **確認文件存在且內容正確**（QA）

## 品質評估標準

| 評估項 | 標準 |
|--------|------|
| **資訊完整性** | 是否回答了核心問題 |
| **具體程度** | 有數據/版本號/步驟 vs. 模糊描述 |
| **時效性** | 日期是否符合需求 |

→ 以上任一不符合 → 回退 WebSearch 或補充搜尋

---

## 注意事項

- gemini cli 路徑：`/opt/homebrew/bin/gemini`
- 輸出過長時可追加：`，精簡回覆，不超過 500 字`
- Gemini 模式需使用 `--yolo` 才能自動操作文件系統
- 搜尋結果需整理進知識庫時，參考 `obsidian-note-writer` skill
