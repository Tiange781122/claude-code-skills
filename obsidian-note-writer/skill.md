# Obsidian 筆記撰寫規範

在 Obsidian 知識庫中撰寫技術筆記時，遵循此規範確保一致性與品質。

## 何時使用

- 新增技術筆記或文件卡片
- 編輯既有筆記內容
- 需要確認撰寫格式是否正確

## 命名規範

**檔名格式**：`YYYY-MM-DD_口語化標題.md`
- 範例：`2025-12-04_FortiGate SSL-VPN 配置指南.md`
- 例外（無需日期前綴）：`CLAUDE.md`、`TODO/TODO.md`、`12.資源/12.1.維護文檔/` 下的指南
- ✅ 繁體中文、描述性清晰、≤50 字
- ❌ 禁止特殊字元 `/\:*?"<>|`

## Frontmatter

至少 1 個標籤（建議 3-7 個）：
```yaml
---
tags:
  - 品牌/領域    # Fortinet、Palo-Alto
  - 技術類別    # 告警、配置、VPN
  - 具體功能    # 日誌上傳、SSL-VPN
---
```
- 標籤禁止空格（`Learning Mode` ❌ → `Learning_Mode` ✅）
- 標籤中間不能有空格，以底線替代
- 可用字元：英文、中文、數字、`-`、`_`

## 內容結構

**必需項目**：
1. 1 個一級標題，≤4 層級
2. 邏輯結構：概述 → 原理 → 步驟 → 驗證 → 最佳實踐
3. **≥1 個 Mermaid 圖表**（優先 `graph TD`、`stateDiagram`、`classDiagram`）
4. 繁體中文、≥2-3 個參考資源

**標準範本**：
```markdown
# 標題

## 📌 概述
## 🎯 適用情境
## 🔍 技術原理（含 Mermaid 圖表）
## ⚙️ 設定步驟
## ✅ 驗證方式
## 💡 最佳實踐
## 📚 相關參考
```

## Mermaid 語法限制

- ❌ 節點標籤禁用括號 `()` `[]` `{}` → 用 `<br/>` 換行代替
  - 錯：`A["流量 (PBR)"]` → 對：`A["流量<br/>PBR"]`
- ❌ 禁用列表語法 `1. `（數字+點+空格）→ 用 `1、` 或 `1.`（無空格）
- ❌ 連線標籤避免括號 → `-->|em6 1G|` ✅

## Fast Text Color

- ~={red}**red**=~：錯誤/危險 · ~={orange}**orange**=~：警告 · ~={green}**green**=~：成功/推薦
- ~={cyan}**cyan**=~：資訊/提示 · ~={purple}**purple**=~：強調/重點

## 概念圖片

為文章加入知識點概念圖，從官方文件下載架構圖、流程圖、對比圖（非 icon 小圖標）：

1. 用 `curl` 抓取官方文件頁面 HTML，提取 `<img src="...">` 中的圖片 URL
2. 優先來源順序：
   - 官方文件頁面（如 `docs.aws.amazon.com`）中的架構圖、流程圖
   - 官方部落格（如 `aws.amazon.com/blogs/`）中的概念對比圖
   - 官方產品頁面中的 how-it-works 圖
3. 下載圖片到 `12.資源/12.3.圖片資產/`，命名格式：`{廠商}-{服務}-{描述}.png`
4. 用 `file` 指令驗證下載的確實是圖片（非 XML/HTML 錯誤頁面），失敗的立即刪除
5. 每個知識點至少配一張概念圖，用 `> **圖片說明**（來源：[連結]）` 標註後接 `![[圖片名.png]]` 嵌入

```bash
# 從官方文件頁面提取圖片 URL
curl -sL "https://docs.example.com/page.html" | grep -oE 'src="[^"]*\.(png|jpg)[^"]*"'

# 下載並驗證
curl -sL -o "$DEST/圖片名.png" "https://完整URL"
file "$DEST/圖片名.png"  # 確認是 PNG image data，不是 XML/HTML
```
