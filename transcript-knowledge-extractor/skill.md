# 課程逐字稿知識提取與整理

從課程逐字稿中提取知識點，透過網路搜尋驗證補充，下載原廠概念圖，整合為結構化圖文筆記。

## 何時使用

- 使用者提供課程、講座、Podcast 的逐字稿
- 使用者說「調用課程逐字稿 skill」

## 工作流程

### 第一階段：提取與分類

1. 讀完整份逐字稿
2. 提取所有知識點，分為兩類：
   - **事實性知識**：可搜尋驗證的概念、技術、數據、理論
   - **講師觀點**：個人見解、經驗談、主觀判斷（直接從逐字稿整理，不搜尋）

### 第二階段：網路搜尋驗證

- 針對「事實性知識」逐一使用 `web_search` 搜尋驗證與補充
- 搜尋重點：該知識點的定義、原理、應用場景
- 必要時用 `web_fetch` 取得官方文件的詳細內容
- 搜尋結果需附上來源連結

### 第三階段：下載原廠概念圖

針對每個主要知識點，從官方文件下載概念介紹圖（非 icon 小圖標）：

1. 用 `curl` 抓取官方文件頁面 HTML，提取 `<img src="...">` 中的圖片 URL
2. 優先來源順序：
   - 官方文件頁面（如 `docs.aws.amazon.com`）中的架構圖、流程圖
   - 官方部落格（如 `aws.amazon.com/blogs/`）中的概念對比圖
   - 官方產品頁面中的 how-it-works 圖
3. 下載圖片到 `12.資源/12.3.圖片資產/`，命名格式：`{廠商}-{服務}-{描述}.png`
4. 用 `file` 指令驗證下載的確實是圖片（非 XML/HTML 錯誤頁面）
5. 每個知識點至少配一張概念圖，用 `> **圖片說明**（來源：[連結]）` 標註

### 第四階段：整合產出

- 確認目標資料夾，接續既有編號
- 遵循 `obsidian-note-writer` skill 的筆記規範
- 繁體中文撰寫，使用 Bash Heredoc 寫入檔案
- 每個知識點說明清楚，標註資料來源
- 概念圖用 `![[圖片名.png]]` 嵌入，搭配 blockquote 說明
- 講師觀點獨立區塊呈現，標明為講師個人見解

## 產出格式

```markdown
# [課程/講座名稱] 知識點整理

## 📌 概述
簡述課程主題與核心內容

> **總覽圖說明**（來源：[連結]）

![[概覽圖.png]]

## 🔍 知識點整理

### 知識點 1：[名稱]
- **說明**：...

> **圖片說明**（來源：[連結]）

![[概念圖1.png]]

- **來源**：[連結]

### 知識點 2：[名稱]
...

## 💬 講師觀點與見解
- 觀點 1：...
- 觀點 2：...

## 📚 參考資源
- [來源 1](url)
- [來源 2](url)
```

## 圖片下載技巧

```bash
# 1. 從官方文件頁面提取圖片 URL
curl -sL "https://docs.example.com/page.html" | grep -oE 'src="[^"]*\.(png|jpg)[^"]*"'

# 2. 從官方部落格提取圖片 URL（通常在 CDN）
curl -sL "https://example.com/blogs/post" | grep -oE 'src="https://[^"]*\.(png|jpg)[^"]*"'

# 3. 下載並驗證
curl -sL -o "$DEST/圖片名.png" "https://完整URL"
file "$DEST/圖片名.png"  # 確認是 PNG image data，不是 XML/HTML
```

## 注意事項

- 逐字稿可能很長，一次接收，自行分批處理
- 如訊息被截斷，主動請使用者接續貼上
- 知識點以逐字稿內容為準，網路搜尋用於驗證與補充，不替換原意
- 圖片必須是概念介紹圖（架構圖、流程圖、對比圖），不要只放 icon 小圖標
- 下載失敗的圖片（XML/HTML）要立即刪除，不要留在圖片資產中
